## 📱 第一步：准备应用图标（10分钟）

### 1. 转换SVG为PNG

你的图标在这里：`/public/icons/app-icon-1024.svg`

**转换方法**（选一个）：

**方法A - 浏览器截图**（最简单）：
1. 在浏览器中打开 `app-icon-1024.svg`
2. Mac截图：`Cmd + Shift + 4`
3. 拖拽选择整个图标区域
4. 保存为 `icon-1024.png`

**方法B - 在线转换**：
1. 访问 https://cloudconvert.com/svg-to-png
2. 上传 `app-icon-1024.svg`
3. 设置尺寸：1024x1024
4. 下载PNG

### 2. 生成所有尺寸

1. 访问 https://www.appicon.co/
2. 上传你的 `icon-1024.png`
3. 勾选 ☑️ iPhone 和 ☑️ iPad
4. 点击 **Generate**
5. 下载zip文件
6. 解压，得到 `AppIcon.appiconset` 文件夹
7. **保存好这个文件夹，等会在Xcode中用**

📖 详细图标指南：`/public/icons/ICON-GUIDE.md`

## 💻 第二步：安装Capacitor（在Mac上操作）

### 前提条件检查

- [ ] 你在Mac电脑上
- [ ] 已安装Node.js（你应该已经有了）
- [ ] 已安装Xcode（从Mac App Store下载，约12GB）

### 安装命令

打开终端（Terminal），进入你的项目目录，运行：

```bash
# 1. 安装Capacitor核心包
npm install @capacitor/core @capacitor/cli

# 2. 安装iOS平台
npm install @capacitor/ios

# 3. 安装启动屏幕和状态栏插件（可选但推荐）
npm install @capacitor/splash-screen @capacitor/status-bar

# 4. 构建Web应用
npm run build

# 5. 添加iOS平台（会创建 ios/ 文件夹）
npm run cap:add:ios
```

**预计时间**：5-10分钟（取决于网速）

### 如果遇到错误

**❌ "Capacitor configuration not found"**
```bash
# 确保 capacitor.config.ts 存在
ls capacitor.config.ts

# 如果不存在，说明文件没保存好
```

**❌ "dist directory not found"**
```bash
# 先构建项目
npm run build

# 确认dist文件夹存在
ls dist
```

## 📱 第三步：打开Xcode并配置

### 1. 打开Xcode项目

在终端运行：

```bash
npm run cap:open:ios
```

或者：

```bash
npx cap open ios
```

Xcode会自动打开你的项目！

### 2. 配置签名（重要！）

**步骤**：
1. 左侧文件列表 → 点击最顶部的 **App**（蓝色图标）
2. 选择中间的 **Signing & Capabilities** 标签
3. 找到 **Team** 下拉菜单
4. 如果看到你的账号 → 选择它
5. 如果没有看到 → 点击 **Add Account...**
   - 输入你的Apple ID（开发者账号）
   - 输入密码
   - 等待登录完成

**确认**：
- ✅ Team已选择
- ✅ Bundle Identifier显示：`com.mininxt.fatum`
- ✅ 没有红色错误提示

### 3. 添加应用图标

**步骤**：
1. 左侧文件列表 → **App** → **Assets.xcassets** → **AppIcon**
2. 右侧会显示很多空的图标格子
3. 打开你刚才下载的 `AppIcon.appiconset` 文件夹
4. 把里面所有PNG文件拖拽到Xcode的格子里
   - 或者直接用整个文件夹替换

**确认**：
- ✅ 所有格子都有图标
- ✅ 1024x1024的格子已填充
- ✅ 没有黄色警告

### 4. 设置版本号

**步骤**：
1. 确保选中 **App**（蓝色图标）
2. 选择 **General** 标签
3. 找到 **Identity** 部分
4. 设置：
   - **Display Name**: `Mystic Arts`
   - **Version**: `1.0.0`
   - **Build**: `1`

### 5. 设置部署目标

在 **General** 标签：
- **Minimum Deployments** → **iOS**: 选择 `13.0` 或更高

## 📦 第四步：构建和上传

### 1. 选择目标设备

Xcode顶部工具栏，左边有个设备选择器：
- 点击它
- 选择 **Any iOS Device (arm64)**
- 不要选择模拟器！

### 2. Archive（打包）

**步骤**：
1. Xcode顶部菜单 → **Product** → **Archive**
2. 等待构建...（首次需要5-15分钟）
3. 构建成功后，会自动打开 **Organizer** 窗口

**如果构建失败**：
- 检查是否选择了"Any iOS Device"
- 检查Team是否正确
- 菜单栏 → Product → Clean Build Folder
- 重试

### 3. 验证（Validate）

在 **Organizer** 窗口：
1. 选择你的Archive（版本1.0.0 (1)）
2. 点击右侧的 **Validate App** 按钮
3. 选择：**Automatically manage signing**
4. 点击 **Next** → **Validate**
5. 等待验证（1-5分钟）

**确认**：
- ✅ 显示绿色勾 "App passed validation"
- ✅ 没有错误

### 4. 上传到App Store Connect

验证通过后：
1. 点击 **Distribute App**
2. 选择：**App Store Connect** → Next
3. 选择：**Upload** → Next
4. 选择：**Automatically manage signing** → Next
5. 选择：**Upload** → Next
6. 等待上传（5-15分钟）

**成功标志**：
- ✅ 显示 "Upload Successful"
- ✅ 收到Apple的邮件确认
